Web Apps and Databases

What are Web Applications

In the most simplest terms, a web application is a program that is used without having to download the program onto your system.
Usually, a user would have to install an application or program onto their system in order to use it effectively; however, a web app
allows for a user to run the software directly from the web using a browser. But what the benefits are for a web app vs traditional apps.

  • Platform Independence

    • With a web app, we don't have to account for all the various different platforms/OS's as the software is compatible to the browser vs the host machine

  • One universal Update

    • Rather than having to update all unique hosts that have the software installed, we update the servers that run the software. This allows for quick fixes.

  • Accessibility

    • Easily accessible in that any device that has access to a browser will also have access to the web app software.

Compared to traditional web sites. Web Apps do more than just provide information to the user. The most important feature is that they are interactive.
Most web apps are comprised of the following:

  • Database: Stores and retrieves necessary information
  • Web Server: handles requests coming from the client
  • Application Server: Completes the requested task

How Web Applications work with Databases

As stated earlier, the web application handles the client request. Once a request is made by the user from their system,
the database will either retrieve whatever information is queried or store the information entered by the user.

Intranet vs Internet

Though there are a lot of similarities between internet networks and intranet networks
Internet networks are global networks that allow any user to access their site.
Although both leverage the internet protocols and world wide web standards, intranets are
private networks that limit access to only certain individuals or groups. The intent is for
them to be private from the global or everyday users.
